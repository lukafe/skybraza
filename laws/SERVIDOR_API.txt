Arranque do servidor web (API + SPA)
====================================

Na raiz do projeto Certik_VASP:

  uvicorn main:app --reload --host 127.0.0.1 --port 8000

Abrir no navegador: http://127.0.0.1:8000/

Deploy Vercel: zero-config FastAPI — main.py na raiz.
Em pyproject.toml, secção [project] deve incluir `dependencies` com fastapi etc.
(senão a Vercel pode não instalar pacotes e o import de main.py falha).
Ver: https://vercel.com/docs/functions/runtimes/python#dependencies
Em local também: npx vercel dev

Estáticos: após editar public/, execute python scripts/sync_vercel_public.py e faça commit de vercel_public/
(cópia usada pelo bundle Python na Vercel).

Endpoints JSON (versão REST recomendada: /api/v1/*; /api/* espelha o mesmo)
----------------------------------------------------------------------------
  GET  /api/v1/health
       — status, api_schema_version (atual: 3), api_rest_version, features.custodiante_track, features.corretora_track

  GET  /api/v1/questions?track=intermediaria|custodiante|corretora
       — blocos e perguntas da trilha; Cache-Control público (5 min)

  POST /api/v1/scope
       — corpo JSON:
         {
           "institution": "...",
           "answers": { ... },
           "track": "intermediaria"   // opcional; default intermediaria; ou "custodiante" | "corretora"
         }
       — resposta inclui track, incisos sujeitos / fora de escopo, resumo, corpus_readiness, journey_2

Trilhas custodiante e corretora (feature flags)
-----------------------------------------------
  Por omissão ambas estão ativas.
  Para desativar custodiante (API devolve 403 em track=custodiante):
    CERTIK_ENABLE_CUSTODIANTE_TRACK=0   (ou false, no, off)
  Para desativar corretora (API devolve 403 em track=corretora):
    CERTIK_ENABLE_CORRETORA_TRACK=0   (ou false, no, off)

CORS
----
  Por omissão: allow_origins = * (adequado a demos).
  Produção: defina CORS_ALLOWED_ORIGINS com lista separada por vírgulas, ex.:
    CORS_ALLOWED_ORIGINS=https://app.exemplo.com,https://www.exemplo.com

Rate limit
----------
  Configurável por env (ver rate_limit.py). Em serverless o limite por IP pode ser pouco eficaz entre
  invocações; para APIs expostas use CERTIK_API_KEY e/ou gateway à frente.

Segurança opcional (API key)
----------------------------
Se definir a variável de ambiente CERTIK_API_KEY, todos os pedidos a /api/*
exceto GET /api/health e GET /api/v1/health devem enviar o cabeçalho:

  X-Certik-Api-Key: <mesmo valor>

Útil para staging ou rede interna. Em desenvolvimento local, omita a variável.

Narrativas «por que será auditado» (opcional)
---------------------------------------------
Com a variável de ambiente GEMINI_API_KEY definida, o motor pode reescrever as explicações
por inciso com o modelo configurado em GEMINI_MODEL (padrão: gemini-1.5-flash).
Sem chave, usam-se textos estáticos em português do Brasil.

Pentest (Jornada 2)
-------------------
O URL do Google Forms de scoping de pentest vem de `laws/AUDITOR_EVIDENCE_BY_INCISO.yaml`
(`pentest_form_url_default`) ou da variável `CERTIK_PENTEST_FORM_URL` se definida.

-----------------
